# Knowledge Gap Formulation (Week 12)

This repository captures one full knowledge-gap exercise: from a technical question, to an explainer, to communication artifacts (thread, summaries, signoff, and sources).

## What this repo is for

- Document how a technical question was sharpened and answered.
- Package the answer in both long-form (`explainer.md`) and social format (`thread.md`).
- Record collaboration checkpoints (morning summary, evening summary, signoff).

## Repository layout

- `README.md`  
  Short guide to the repository and file purposes.

- `Hiwot_DAY_1/question.md`  
  The original technical question framing the gap.

- `Hiwot_DAY_1/explainer.md`  
  Main answer document: concepts, profiling approach, metrics, and practical fix.

- `Hiwot_DAY_1/thread.md`  
  A 6-post X thread version of the explainer.

- `Hiwot_DAY_1/sources.md`  
  Canonical references used in the explainer plus the profiling pattern/tool used.

- `Hiwot_DAY_1/morning_call_summary.md`  
  3-5 sentence note on what was ambiguous and how the question was sharpened.

- `Hiwot_DAY_1/evening_call_summary.md`  
  3-5 sentence note on feedback received and what was revised.

- `Hiwot_DAY_1/signoff.md`  
  Asker's gap-closure judgment and what became clear after the answer.

- `Hiwot_DAY_1/grounding_commit.md`  
  Grounding note linking the discussion to the concrete code change/commit context.
