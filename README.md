# Knowledge Gap Formulation (Week 12)

This repository captures a four-day paired gap-closing cycle: each day includes a sharpened technical question, a deep explainer, communication artifacts, and a grounding commit that ties the learning to a concrete project change.

## Repository Purpose

- Show how ambiguous technical questions were refined into testable engineering claims.
- Preserve both long-form explainers and public-facing communication artifacts.
- Record peer calibration checkpoints (morning call, evening call, signoff).
- Document grounding commits that improve Weeks 10 and 11 portfolio artifacts.

## Repository Layout

- `README.md`  
  Repository guide and public artifact links.

- `Hiwot_DAY_1/`, `Eyoel_DAY_2/`, `Nahom_DAY_3/`, `Mamaru_DAY_4/`  
  Each day folder contains:
  - `question.md`
  - `explainer.md`
  - `thread.md`
  - `sources.md`
  - `morning_call_summary.md`
  - `evening_call_summary.md`
  - `signoff.md`
  - `grounding_commit.md`

- `synthesis.md`  
  Week synthesis covering 8 closed gaps, key learning, and canonical contributions.

- `canonical_list.md`  
  Annotated papers/tools/patterns contributed to the cohort canon.

- `portfolio_update.md`  
  Hiring-manager-facing summary of how grounding commits strengthen Weeks 10 and 11 artifacts.
