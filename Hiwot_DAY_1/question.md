# Question: attention sinks and our LLM message layout

In `agent/services/enrichment/llm.py`, `OpenRouterJSONClient.generate_model` constructs a two-message chat where the system message is a single concatenated string (the caller’s `system_prompt` plus fixed “valid JSON only / no markdown / don’t invent evidence” instructions) placed before the user message, which is the full `json.dumps(user_payload)` and can be quite large. In streaming / long-context regimes, people discuss attention sinks and the disproportionate influence of early tokens. Mechanistically, why might the first lines of that system prompt shape later compliance (e.g., JSON-only, no markdown) more than a late reminder buried after a large or noisy user JSON—even though the model “sees” the entire context in one forward pass during prefill? What’s one prompting or transcript-layout mitigation you would apply directly in how these messages are constructed in this code, and what tradeoff does it introduce?
